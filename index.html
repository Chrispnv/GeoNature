GEONATURE

Une application opensource de gestion des données naturalistes. Créée par les parcs nationaux de France.

En savoir plus

A propos

Application de synthèse des observations faune et flore.

Elle regroupe toutes les données des différents protocoles FAUNE et FLORE en les limitant au niveau QUI QUOI QUAND OU.

Principe général : Un protocole = Un outil = Une BDD + Une BDD et une application de SYNTHESE regroupant les données des différents protocoles sur la base des champs communs à tous les protocoles (QUI a vu QUOI, OU et QUAND).

A REVOIR car il ne s'agit pas uniquement de la SYNTHESE

Code source, suivi des versions et des tickets sur https://github.com/PnEcrins/GeoNature

Documentation (serveur, applications, base de données) sur http://geonature.rtfd.org

Testez l'application ! test.com

1- Les données sources de chaque protocole

  • Chaque protocole dispose d'un schéma dédié dans la BDD.
  • Il est possible de créer autant de protocoles (et donc de schémas) que souhaité.
  • Les tables de chaque schéma des protocoles correspondent aux besoins et aux champs spécifiques de ceux-ci.
  • Ces tables des protocoles sont alimentées directement par des applications web ou mobiles ou indirectement par des sources externes (tableurs, BDD, SIG, GPX...), éventuellement automatisés par un ETL (Extract Transform Load comme Talend).
  • L'application est fournie avec les schémas et applications web des protocoles Contact faune vertébrés, invertébrés et mortalité.
  • A chaque fois qu'une donnée (observation) est insérée, modifiée ou supprimée dans une table d'un des protocoles, un trigger (déclencheur automatique dans la BDD) lance une requête SQL venant mettre à jour le schéma de synthèse.
  • Les protocoles sont listés dans la table bib_sources du schéma synthese.

2 - La synthèse des données de tous les protocoles

  • Elle regroupe les données de tous les protocoles avec uniquement leurs champs communs "Qui a vu Quoi Où et Quand + Comment ?" (Observateur, taxon, localisation, date, protocole).
  • Elle est alimentée automatiquement à chaque fois qu'une donnée est saisie dans le schéma d'un protocole.
  • L'application fournie permet de consulter l'ensemble de ces données, de les filtrer (par taxon, statut, date, période, observateur, source, commune, secteur, zone réglementaire ou dessinée), de les exporter (tableur ou SIG) ou d'éditer la donnée source si une application web est disponible pour le protocole source.

3 - Export partenaires

  • L'application permet de préparer et d'automatiser les exports à destination des partenaires (SINP, INPN, LPO...).
  • Pour chaque partenaire, une vue spécifique est créée, permettant de définir les données à rendre accessible.
  • Chaque partenaire peut ainsi récupérer les données à jour quand ils le souhaitent et en toute autonomie.
  • A l'inverse il est possible d'intégrer dans la synthèse les données provenant de partenaires pour pouvoir les consulter au côté des données provenent des protocoles internes.

4 - TAXREF

  • Toutes les données s'appuient sur le référentiel taxonomique TAXREF réalisé dans le cadre de l'INPN.
  • En se basant sur TAXREF pour identifier chaque espèce, toutes les observations sont rattachées à un identifiant d'espèce unique et commun au niveau national.

2012

Le Parc national des Ecrins modernise et réorganise ses bases de données faune et flore autour de la synthèse.

Il développe alors les applications CONTACT FAUNE vertébrés, invertébrés et mortalité ainsi que l'application SYNTHESE.

Début 2014

L'ensemble des parcs nationaux retient ce principe pour l'organisation de ses bases de données faune et flore.

...

Décembre 2014

Gil Deluermoz et Amandine Sahl, développeurs du PnEcrins et PnCévennes, se réunissent pour travailler une semaine sur la refonte de l'applicatin pour la rendre plus générique et ainsi faciliter son déploiement dans les différents parcs nationaux mais aussi dans d'autres structures souhaitant les utiliser.

Ils publient l'application sur la forge Github, y font de nombreuses évolutions aboutissant à la version 1.1.0, documentent le processus d'installation et proposent une roadmap.
